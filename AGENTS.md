## Codex / Claude 共同同步规则

1. 开始修改前，先阅读 `docs/decision-log.md`、`docs/progress.md`、`docs/handoff.md` 和 `docs/git-log.md`
2. 写文章前，先阅读 `docs/author-pool.md`，根据主题选择合适的虚拟作者
3. 写内容发布流程前，先阅读 `docs/workflow.md`
4. 每次完成阶段性工作后，更新对应文档，必要时同时更新多个文件
5. 关键决策写入 `docs/decision-log.md`
6. 当前进度写入 `docs/progress.md`
7. 交接摘要写入 `docs/handoff.md`
8. 重要 commit / push 记录写入 `docs/git-log.md`
9. 如果策略、优先级或实现方案变化，先更新文档再继续开发
10. `CLAUDE.md` 和 `AGENTS.md` 需要保持同一套内容策略与同步规则，不要各写各的
